IC Report App · Documento de referência para desenvolvimento via Vibe Coding · Maio 2026

1. Objetivo deste Documento

Este documento lista todas as variáveis que o app precisa popular dinamicamente ao receber uma planilha Excel do OptimaReports e gerar o report HTML (landing page ou dashboard). Cada variável é mapeada ao seu row index no CSV, ao tipo de dado e ao exemplo real do AlphaFranchise Abril 2026.

✅ Arquitetura recomendada: Python (parse Excel → dict de variáveis) + Jinja2 (dict → HTML renderizado). O HTML do dashboard usa um objeto const D = {...} — basta substituir pelo dict gerado.

7. Listas Dinâmicas (calculadas pelo parser)

Estas variáveis são listas ordenadas geradas pelo parser — não lidas diretamente do CSV.

KBP Alerts — top N por KBP (dinâmico por mês/franquia)

kbp1_alerts: [[label, key, total], ...]   # CE — todos com val>0, sorted desc
kbp2_alerts: [[label, key, total], ...]   # RC — top 8 com mais incidências
kbp3_alerts: [[label, key, total], ...]   # CO — todos com val>0, sorted desc

kbp1_all:  [[label, total], ...]          # CE — para resumen lateral
kbp2_all:  [[label, total], ...]          # RC — para resumen lateral
kbp3_all:  [[label, total], ...]          # CO — para resumen lateral

kbp1_store_ranks: {key: [[store, val], ...top5]}   # por alerta CE
kbp2_store_ranks: {key: [[store, val], ...top5]}   # por alerta RC
kbp3_store_ranks: {key: [[store, val], ...top5]}   # por alerta CO

8. Lógica de Cálculo Crítica

Barras proporcionais (CRÍTICO)

# Sempre normalizar pelo máximo da lista, NÃO pelo total
max_val = max(all_values_in_list)
bar_width_pct = (val / max_val) * 100

# Exemplo: barras de Top5 Tipos de Alerta
max_alert = top5_alerts[0]['val']  # maior valor
for alert in top5_alerts:
    width = alert['val'] / max_alert * 100  # % relativo

KBP RC — top 8 dinâmico

# RC tem 31 alertas — exibir apenas top 8 com mais incidências
RC_ALL_KEYS = ['mobile','outside','cash_drawer','open_late',
               'inactive','playing','receipt_given','receipt_print',
               'check_packaging','dt_pin','dt_outside','dt_cash',
               'dt_mobile','dt_receipt', ...]  # 31 total

kbp2_all_raw = [(label, key, val) for label,key in RC_ALL_KEYS if val > 0]
kbp2_all = sorted(kbp2_all_raw, key=lambda x: -x[2])
kbp2_alerts = kbp2_all[:8]  # top 8 para os painéis

Cam Check — ranking de loja

# Ordenar all_alerts desc → posição = índice+1
all_ranked = sorted(stores, key=lambda s: -s['all_alerts'])
rank_map = {store: i+1 for i, store in enumerate(all_ranked)}
# Loja #1 = mais alertas (pior), #62 = menos alertas

10. Checklist para o Agente de IA

  • ✅ Row indices do CSV são fixos — usar o mapa da seção 3–6
  • ✅ Col 1 = total da rede (não somar lojas individualmente)
  • ✅ Nomes de loja = row 5, cols 2..N — dinâmico, varia por franquia
  • ⚠️ delivery_wait (row 79) pertence ao KBP CO — não RC
  • ✅ RC tem 31 alertas — exibir apenas top 8 por incidência
  • ✅ Filtrar sempre alertas com val > 0 antes de renderizar painéis
  • ✅ Remover sufixos [z], [g] dos nomes de loja na exibição
  • ✅ Barras: normalizar pelo max da lista, nunca pelo total
  • ✅ Cam stores: ordenar por cam desc, limitar a ~13 linhas na tabela
  • ✅ Verificar existência do arquivo antes de processar (franquias ausentes)
  • ✅ Áreas usam cores independentes dos KBPs (cashier≠ce, allstore≠rc)
  • ✅ PDF via window.print() + CSS @media print — sem dependências externas
